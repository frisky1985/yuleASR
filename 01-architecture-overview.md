### 1.2 推荐架构: 模式A - ETH作为DDS传输层

**理由:**
1. **标准兼容**: DDS-RTPS协议原生支持UDP/IP传输，可直接运行于以太网
2. **性能优化**: 减少协议转换开销，端到端延迟更低
3. **生态成熟**: Fast DDS、Cyclone DDS等主流实现均支持ETH传输
4. **运维简单**: 统一网络管理，无需额外网关设备

## 3. 核心组件说明

| 组件 | 职责 | 关键技术 |
|------|------|----------|
| DDS Domain Participant | 域参与者管理，主题发现 | Fast DDS / Cyclone DDS |
| RTPS Protocol | 实时发布订阅协议实现 | RTPS 2.5标准 |
| Transport Adapter | 传输层适配，支持UDP/TCP | DDS-RTPS传输配置 |
| QoS Manager | 服务质量策略管理 | DDS QoS策略集 |
| Config Service | 配置集中管理与同步 | XML/JSON配置 + 发现服务 |
| Monitor Service | 运行时监控与诊断 | DDS监控Topic |
