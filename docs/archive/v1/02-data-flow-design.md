## 2. 数据流分类

### 2.1 实时数据流 (Real-time Stream)

```
┌───────────────────────────────────────────────────────────────────────────┐
│                    实时数据流 (Best-Effort QoS)                        │
├───────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│   ┌───────────────┐                               ┌───────────────┐        │
│   │   Sensor 1    │                               │   Dashboard   │        │
│   │  ┌───────┐    │                               │   ┌───────┐   │        │
│   │  │Temp│─────────────────────────────────────▶│  │Disp│   │        │
│   │  └───────┘    │      ┌───────────────┐     │   └───────┘   │        │
│   │             │      │ Topic:Sensor  │     │             │        │
│   │  ┌───────┐    │─────▶│ Type:TempData │─────▶│             │        │
│   │  │Hum  │─────────────────────────────────────▶│  ┌───────┐   │        │
│   │  └───────┘    │      │ QoS: BE     │     │  │Store│   │        │
│   └───────────────┘      │ Transport:UDP │     │  └───────┘   │        │
│                        └───────────────┘     └───────────────┘        │
│                                                                          │
│   特性:                                                                  │
│   • 低延迟 (<1ms), 无确认机制                                           │
│   • 多播/ 组播传输                                                    │
│   • 适用于传感器数据、实时监控流                                             │
│                                                                          │
└───────────────────────────────────────────────────────────────────────────┘
```

## 3. QoS策略映射

```
┌───────────────────────────────────────────────────────────────────────────┐
│                   QoS策略到ETH参数映射                                  │
├───────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│   DDS QoS Policy          ETH网络参数                                    │
│   ═──────────────═────────────────────────────────────────────────┤
│                                                                          │
│   RELIABILITY             UDP单播/组播 vs TCP                            │
│   ├─ Best Effort         └─ UDP, 组播地址: 239.255.x.x                    │
│   └─ Reliable           ─├ TCP, 单播                                   │
│                          └ 可选: 多播 + 应用层确认                      │
│                                                                          │
│   DURABILITY              缓存策略与内存使用                             │
│   ├─ Volatile           ─└ 无缓存, 消息仅传送给当前活跃订阅者               │
│   ├─ Transient Local    ─└ 本地缓存, 最新值保持在发布者内存               │
│   └─ Transient          ─└ 外部缓存服务, 支持进程持久化                  │
│                                                                          │
│   LATENCY_BUDGET          网络优先级调度                                  │
│   └─ 基于延迟预算         └ DSCP标记, VLAN PCP优先级                     │
│                                                                          │
│   TRANSPORT_PRIORITY      QoS与VLAN映射                                  │
│   └─ 0-255              ─└ VLAN PCP 0-7, 请求级区分                     │
│                                                                          │
│   LIFESPAN                时间敏感网络调度                                  │
│   └─ 过期时间            ─└ 数据包TTL设置, 时间触发传输                     │
│                                                                          │
└──────────────────────────────────────────────────────────────────────────┘
```

## 6. 数据流性能指标

| 数据流类型 | 延迟要求 | 带宽需求 | 可靠性 | ETH配置建议 |
|----------|----------|----------|--------|--------------|
| 传感器数据 | <1ms | 10-50Mbps | BE | 组播优先级 |
| 控制信号 | <5ms | 1-5Mbps | Reliable | 单播+DSCP |
| 状态同步 | <10ms | 5-20Mbps | Persistent | 多播 |
| 配置数据 | <100ms | 0.1-1Mbps | Reliable | TCP单播 |
| 媒体流 | 低延迟 | 100-500Mbps | BE | 组播+VLAN |
