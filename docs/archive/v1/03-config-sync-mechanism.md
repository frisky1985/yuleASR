## 4. 配置数据模型

```yaml
# DDS配置Topic数据结构
define configuration_data:
  # 元数据
  header:
    config_id: string      # 配置唯一标识
    version: string        # 版本号，比如 "1.2.3"
    timestamp: int64       # 发布时间戳
    checksum: string       # SHA256校验和
  
  # 目标节点
  target:
    node_filter:           # 节点过滤条件
      node_ids: []string   # 指定节点ID列表
      node_types: []string # 指定节点类型
      domains: []int32     # 指定Domain ID
  
  # 配置内容
  payload:
    config_type: enum      # CONFIG/PARAMETER/FEATURE
    content: bytes         # 配置内容(Protobuf/JSON)
    compression: enum      # NONE/ZLIB/LZ4
    
  # 生命周期
  lifecycle:
    activate_time: int64   # 生效时间
    expire_time: int64     # 过期时间
    rollback_on_fail: bool # 失败时是否回滚
```

│                                                                          │

│       │  ────────────────────────────────▶│                        │                │
│       │                        │                        │                │
│       │  2. Discovery Response    │                        │                │
│       │  ◀─────────────────────────────────────────┘                        │                │
│       │                        │                        │                │
│       │  ──────────────────────────────────────────────────────▶│                │

│   - 发现服务基于Simple Discovery Protocol (SDP)                            │

## 6. 配置同步策略

| 策略 | 适用场景 | DDS QoS配置 | 优缺点 |
|------|---------|-----------|--------|
| 推模式 (Push) | 小型网络，配置变更少 | Reliable + Durability | 实时性好，节点无需轮询 |
| 拉模式 (Pull) | 大型网络，控制精确 | Request/Reply模式 | 控制可控，服务器压力小 |
| 混合模式 (Hybrid) | 通用场景 | Reliable Topic + 心跳 | 兼顾实时性与可靠性 |
