## 2.1 DDS核心概念

### 2.1.1 什么是DDS

DDS (Data Distribution Service) 是OMG制定的一种以数据为中心的发布/订阅中间件标准。它专为实时、高可靠、高性能的数据传输设计，广泛应用于航空、汽车、工业控制等领域。

### 2.1.2 DDS架构

```
┌─────────────────────────────────────────────────────────────────┐
│                     DDS Domain                                          │
├─────────────────────────────────────────────────────────────────┤
│  ┌────────────────────┐        ┌────────────────────┐       │
│  │   Participant 1   │        │   Participant 2   │       │
│  │  ┌─────────────┐      │        │  ┌─────────────┐      │
│  │  │   Publisher   │────────────────────├───────├Reader─┐ │      │
│  │  │      │        │      │        │      │        │      │
│  │  │   Writer    ────────────────────┼───────├Data──┤ │      │
│  │  │      │        │      │        │      │        │      │
│  │  │   Topic     ────────────────────┴───────└Reader─┘ │      │
│  │  └─────────────┘      │        │  └─────────────┘      │
│  └────────────────────┘        └────────────────────┘       │
└─────────────────────────────────────────────────────────────────┘
```

### 2.1.3 关键实体

| 实体 | 描述 | 功能 |
|------|------|------|
| Domain | 逻辑分隔空间 | 将通信节点分隔为独立的组 |
| Participant | 参与者 | 应用程序进入DDS域的入口点 |
| Topic | 主题 | 数据的逻辑名称和类型定义 |
| Publisher | 发布者 | 管理数据发布 |
| Subscriber | 订阅者 | 管理数据订阅 |
| DataWriter | 数据写入器 | 向主题写入数据 |
| DataReader | 数据读取器 | 从主题读取数据 |

### 2.1.4 QoS (Quality of Service)

DDS通过QoS策略控制通信行为:

```c
// 示例: 可靠发送配置
dds_qos_t *qos = dds_create_qos();
dds_qset_reliability(qos, DDS_RELIABILITY_RELIABLE, DDS_SECS(1));
dds_qset_durability(qos, DDS_DURABILITY_TRANSIENT_LOCAL);
dds_qset_history(qos, DDS_HISTORY_KEEP_LAST, 10);
```

| QoS策略 | 说明 | 适用场景 |
|---------|------|---------|
| Reliability | 可靠性 | Reliable(重要数据) / BestEffort(实时数据) |
| Durability | 持久性 | Volatile / TransientLocal / Persistent |
| History | 历史策略 | KeepLast(n) / KeepAll |
| Deadline | 截止时间 | 定期数据发布要求 |
| LatencyBudget | 延迟预算 | 允许的传输延迟 |
| Lifespan | 生存期 | 数据的有效期 |
| Ownership | 所有权 | Exclusive(主备份) / Shared |

## 2.2 RTPS协议

### 2.2.1 协议概述

RTPS (Real-Time Publish Subscribe) 是DDS的网络层实现协议，定义了:
- 节点发现机制
- 数据交换格式
- 可靠传输机制

### 2.2.2 发现机制

```
┌─────────────────────────────────────────────────────────────────┐
│                   RTPS Discovery Process                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                         │
│  Participant A                         Participant B                   │
│       │                                     │                           │
│       │  1. SPDP Announce (multicast)      │                           │
│       │────────────────────────────────────────┾                           │
│       │                                     │                           │
│       │  2. SPDP Response                  │                           │
│       │────────────────────────────────────────┾                           │
│       │                                     │                           │
│       │  3. SEDP Exchange (endpoints)       │                           │
│       │────────────────────────────────────────┾                           │
│       │                                     │                           │
│       │  4. Data Flow Established          │                           │
│       │────────────────────────────────────────┾                           │
│                                                                         │
└─────────────────────────────────────────────────────────────────┘
```

## 2.5 TSN基础

TSN (Time-Sensitive Networking) 提供确定性低延迟的以太网传输:

| 标准 | 功能 | 用途 |
|-----|------|------|
| 802.1Qbv | 增量门控制 | 保证关键数据的确定性时段 |
| 802.1Qbu | 帧披露 | 降低非关键数据的延迟 |
| 802.1CB | 帧复制和可靠性消除 | 提高可靠性 |
| 802.1AS | gPTP时间同步 | 精确时间同步 |
| 802.1Qca | 路径控制和居留 | 确定性路径分配 |
