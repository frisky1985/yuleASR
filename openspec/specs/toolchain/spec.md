# OpenSpec: Yule Toolchain 工具链规范

> **版本**: v1.0  
> **状态**: 活跃  
> **最后更新**: 2026-04-13

## 概述

Yule Toolchain 是予乐科技为汽车基础软件开发提供的一站式工具链，包含配置工具、代码生成器、编译构建工具和调试诊断工具。

## 工具链架构

```

├─────────────┬─────────────┬─────────────┬─────────────────┤
│  Configurator│  Generator  │   Builder   │    Debugger     │

│  企标导入   │  模板引擎   │  依赖管理   │   实时监控      │
│  参数校验   │  代码生成   │  CI/CD集成  │   诊断通信      │
│  版本管理   │  文档生成   │  版本控制   │   刷写工具      │
```

## 1. Yule Configurator - 在线配置工具

### 功能规范

#### 1.1 可视化配置

**Requirements**:
- REQ-CFG-001: 提供图形化界面配置基础软件参数
- REQ-CFG-002: 支持模块拖拽式添加和删除
- REQ-CFG-003: 参数配置实时校验
- REQ-CFG-004: 配置变更历史记录

**Scenarios**:
- ✅ 用户通过拖拽添加 CAN 模块
- ✅ 用户配置 CAN 波特率为 500Kbps
- ✅ 系统实时校验参数合法性
- ✅ 用户回滚到上一个配置版本

#### 1.2 企标导入

**Requirements**:
- REQ-CFG-010: 支持 OEM 企业标准文件导入
- REQ-CFG-011: 支持 Excel/JSON/XML 格式
- REQ-CFG-012: 自动解析企标并生成配置
- REQ-CFG-013: 企标差异对比功能

**Scenarios**:
- ✅ 导入某 OEM 的 CAN 矩阵表 (Excel)
- ✅ 解析并生成 Com 模块配置
- ✅ 对比新旧企标差异

#### 1.3 模块选择

**Requirements**:
- REQ-CFG-020: 按需选择基础软件模块
- REQ-CFG-021: 模块依赖自动解析
- REQ-CFG-022: 模块冲突检测
- REQ-CFG-023: 资源占用预估

**支持的模块列表**:
```yaml
modules:
  - name: Mcu
    description: 微控制器驱动
    dependencies: []
  - name: Port
    description: GPIO驱动
    dependencies: [Mcu]
  - name: Can
    description: CAN通信驱动
    dependencies: [Mcu, Port]
  - name: Com
    description: 通信服务
    dependencies: [Can, PduR]
  - name: Dcm
    description: 诊断服务
    dependencies: [Com, PduR]
  - name: NvM
    description: 存储服务
    dependencies: [MemIf, Fls]
```

**模板示例**:


### 技术规范

#### 生成器架构

```python
# 核心类设计
class ArxmlParser:
    """ARXML 解析器"""
    def parse(self, file_path: str) -> ArxmlModel:
        pass
    
    def validate(self, model: ArxmlModel) -> ValidationResult:
        pass

class CodeGenerator:
    """代码生成器"""
    def __init__(self, template_dir: str):
        self.jinja_env = Environment(loader=FileSystemLoader(template_dir))
    
    def generate(self, model: ArxmlModel, output_dir: str) -> GenerationResult:
        """生成代码"""
        for module in model.modules:
            self._generate_module(module, output_dir)
    
    def _generate_module(self, module: Module, output_dir: str):
        template = self.jinja_env.get_template(f"{module.name}.j2")
        code = template.render(module=module)
        self._write_file(output_dir, module.name, code)

class IncrementalGenerator(CodeGenerator):
    """增量生成器"""
    def generate_incremental(self, model: ArxmlModel, 
                            last_model: ArxmlModel,
                            output_dir: str) -> GenerationResult:
        """只生成变更的部分"""
        diff = self._calculate_diff(model, last_model)
        for change in diff.changes:
            self._generate_change(change, output_dir)
```

## 3. Yule Builder - 编译构建工具

### 功能规范

#### 3.1 编译器集成

**支持的编译器**:
| 编译器 | 版本 | 平台 |
|:-------|:-----|:-----|
| GCC ARM | 10.3+ | ARM Cortex-M |
| IAR EWARM | 8.5+ | ARM |
| Tasking | 6.3+ | TriCore |
| Green Hills | 2019+ | ARM/PowerPC |

**Requirements**:
- REQ-BLD-001: 自动检测编译器
- REQ-BLD-002: 生成编译脚本
- REQ-BLD-003: 支持交叉编译
- REQ-BLD-004: 编译优化选项配置

#### 3.2 构建脚本

**生成的 Makefile 示例**:
```makefile
# Generated by Yule Builder

CC = arm-none-eabi-gcc
CFLAGS = -mcpu=cortex-m7 -mthumb -O2 -g
INCLUDES = -I./include -I./generated/include

SOURCES = $(wildcard ./src/*.c) $(wildcard ./generated/src/*.c)
OBJECTS = $(SOURCES:.c=.o)
TARGET = app.elf

.PHONY: all clean

all: $(TARGET)

$(TARGET): $(OBJECTS)
	$(CC) $(OBJECTS) -o $@ $(LDFLAGS)

%.o: %.c
	$(CC) $(CFLAGS) $(INCLUDES) -c $< -o $@

clean:
	rm -f $(OBJECTS) $(TARGET)
```

#### 3.3 CI/CD 集成

**支持的 CI 平台**:
- Jenkins
- GitLab CI
- GitHub Actions

**Pipeline 示例**:
```yaml
# .gitlab-ci.yml (generated)
stages:
  - build
  - test
  - report

build:
  stage: build
  script:
    - yule-builder generate
    - yule-builder compile --compiler gcc
  artifacts:
    paths:
      - build/

test:
  stage: test
  script:
    - yule-builder test --coverage
  coverage: '/TOTAL.*\s+(\d+%)$/'
```
